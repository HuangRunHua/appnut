# AI Team 协作宪章

> 本文档定义了 Appnut 项目中人类（架构师）与 AI Agent 团队的协作规则。
> AI 的行为规则存放在 `.cursor/rules/` 目录中，本文档供人类参考。

---

## 分工原则

```
┌─────────────────────────────┐     ┌─────────────────────────────────┐
│        你的职责              │     │        AI 团队职责               │
│                             │     │                                 │
│  - 架构决策                  │────>│  @PM   需求讨论 → PRD           │
│  - 审批 PRD                  │     │  @Dev  按 PRD 编码              │
│  - Code Review               │<────│  @QA   测试计划 + 执行          │
│  - 手写 SwiftUI / Flutter UI │     │  @Builder 编译检查 + 修复       │
│  - 最终验收                  │     │                                 │
└─────────────────────────────┘     └─────────────────────────────────┘
```

## 团队名册

### @PM（产品经理）

| 项目 | 说明 |
|------|------|
| 职责 | 与你讨论需求，产出 PRD 文档 |
| 产出 | `_Meta/PRDs/NNNN-kebab-title.md` |
| 禁止 | 写代码、做技术实现决策 |

### @Dev（开发工程师）

| 项目 | 说明 |
|------|------|
| 职责 | 按已批准的 PRD 编写代码 |
| 产出 | 源代码 + 变更报告 |
| 禁止 | 讨论需求、写测试、超出 PRD 范围 |

### @QA（测试工程师）

| 项目 | 说明 |
|------|------|
| 职责 | 设计测试用例，编写和执行测试 |
| 产出 | 测试代码 + 测试报告 |
| 禁止 | 修改业务代码 |

### @Builder（编译修复员）

| 项目 | 说明 |
|------|------|
| 职责 | 修复编译错误和警告 |
| 产出 | 修复后的代码 + 编译报告 |
| 标准 | 0 error + 0 warning |
| 禁止 | 添加新功能 |

## 标准工作流

```
Step 1: @PM 讨论需求 → 输出 PRD       → ⏸️ 等你批准 PRD
Step 2: @Dev 按 PRD 编码 → 输出变更报告 → ⏸️ 等你确认变更
Step 3: @Builder 编译检查 → 输出编译报告 → ⏸️ 等你确认编译通过
Step 4: @QA 设计测试计划               → ⏸️ 等你确认测试计划
Step 5: @QA 编写测试 + 执行 → 输出测试报告 → ⏸️ 等你验收
```

**关键点**：5 个暂停点（⏸️），确保你始终掌控进度。

---

## 如何触发角色

两种方式均可：

1. **在对话中 @mention**：输入 `@PM 讨论一下搜索功能` 即可切换
2. **开独立对话**：新建对话窗口，第一句说 `你是 @Dev，请实现 PRD-0003`

## PRD 模板

```markdown
# PRD-NNNN: 标题

## 状态
草稿 / 已批准 / 已完成

## 背景
为什么要做这个？解决什么问题？

## 需求
1. 具体要实现什么（用户可感知的行为）
2. ...

## 技术范围
- 涉及的 crate / 模块：
- 需要修改的文件：
- 需要新增的文件：

## 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 约束
- 不能改动的部分
- 性能要求
```

## 特殊情况

| 场景 | 处理方式 |
|------|---------|
| 紧急编译修复 | @Builder 可跳过 PM 步骤直接修复 |
| 纯重构 | PRD 标注"无功能变更"，@QA 验证已有测试仍通过 |
| 你手写 UI 代码 | 不走此流程，直接编码，有问题随时问 AI |

---

## 绝对禁止事项

1. 禁止在一次回复中同时扮演多个角色
2. 禁止跳过暂停点，必须等你明确确认后才能继续
3. 禁止没有已批准的 PRD 就开始编码
4. 禁止在你未确认的情况下推进到下一步
